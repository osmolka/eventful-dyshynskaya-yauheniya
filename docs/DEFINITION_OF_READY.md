# Definition of Ready

Use this checklist before starting implementation
or before asking Copilot to plan or write code.

---

## 1. Objective
What do I want to achieve?

- What problem does this task solve?
- Why is it needed *now*?

✅ One or two clear sentences.

---

## 2. Scope
What is included?

- What functionality should be implemented?
- What parts of the system are affected?

### Non-scope
- What is explicitly NOT included?
- What should NOT be changed?

---

## 3. Constraints
Hard rules that must not be violated.

- Tech stack / language versions:
- Libraries that must / must not be used:
- Access restrictions:
- Time / complexity limits (if any):

---

## 4. Relevant code areas
Where should the agent look?

- Files:
- Folders:
- Entry points:
- Existing examples (if any):

---

## 5. Acceptance Criteria
How do I know the task is done?

- [ ] Functional result is achieved
- [ ] Code builds / runs
- [ ] No unrelated changes
- [ ] Documentation updated (if needed)

(Add task-specific criteria.)
